Соглашения по работе с git:

Никогда не коммитить прямо в main/master или develop. Создавать новую ветку для каждой задачи. Делать частые, но атомарные коммиты (один коммит — одно логическое изменение). Перед слиянием (merge) делать git pull --rebase для актуализации ветки.

    main / master: продакшн-код
    dev - ветка разработки
    future/ - ветки фичей
    bugfix/ - ветки исправления багов
